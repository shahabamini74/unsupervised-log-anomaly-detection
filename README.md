# Unsupervised Anomaly Detection in Large-Scale System Logs

This repository contains the implementation of my Master's Thesis project developed during my Digital Transformation Management degree at the University of Bologna.

The project presents an end-to-end machine learning pipeline for detecting anomalies in large-scale distributed system logs using unsupervised learning techniques. The system is validated on the HDFS (Hadoop Distributed File System) log dataset.

---

## Project Overview

Modern distributed systems generate massive volumes of logs that are difficult to analyze manually. Detecting anomalies in these logs is critical for identifying failures, performance degradation, and abnormal system behavior.

This project builds a complete anomaly detection workflow that transforms raw log data into structured features and applies multiple unsupervised machine learning models to identify abnormal execution patterns.

The pipeline focuses on **BlockId-level execution traces**, which represent sequences of log events generated by the same data block in HDFS.

---

## Pipeline Architecture

The anomaly detection workflow consists of the following stages:

1. **Log Parsing**
   - Raw HDFS log messages are parsed using the Drain algorithm.
   - Variable components such as timestamps, IP addresses, and block identifiers are abstracted into log templates.

2. **Session Construction**
   - Log events are grouped by **BlockId** to reconstruct execution traces.
   - Each block trace represents the sequence of events associated with a specific data block.

3. **Feature Engineering**
   - Event templates are converted into a numerical representation using an **event count matrix**.
   - TF-IDF weighting is applied to emphasize rare but informative events while reducing the impact of common background events.

4. **Anomaly Detection Models**

   Multiple unsupervised models are evaluated:

   - **Isolation Forest**
   - **Principal Component Analysis (PCA)**
   - **Incremental PCA (IPCA)**
   - **Cluster-based Detection (K-Means + Isolation Forest)**

5. **Evaluation and Validation**

   Although the models are trained in an unsupervised manner, ground truth labels from the dataset are used for post-hoc evaluation.

   Evaluation metrics include:

   - ROC-AUC
   - Precision
   - Recall
   - F1 Score
   - Confusion Matrix

---

## Dataset

The project uses the **HDFS log dataset**, a widely used benchmark for log anomaly detection research.

Dataset characteristics:

- ~11 million raw log lines
- Block-level identifiers for session reconstruction
- Ground truth anomaly labels for evaluation
- Highly imbalanced distribution (normal events dominate)

## Key Results

The experimental evaluation shows that **Isolation Forest** performs best among the tested models for detecting anomalies in TF-IDF log representations.

Example results:

- ROC-AUC ≈ 0.99
- F1 Score ≈ 0.73
- Effective detection of abnormal block execution patterns

---

## From Statistical Anomaly to Operational Insight

A major goal of the project is connecting statistical anomalies to real operational issues.

Detected anomalies are interpreted as potential indicators of:

- Data node failures
- Block replication issues
- Data corruption
- Metadata stress in the NameNode

This bridges the gap between machine learning outputs and real system operations.
